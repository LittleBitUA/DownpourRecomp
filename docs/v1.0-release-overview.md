# v1.0 release overview

Snapshot of what shipped on 2026-06-27, what hardware it was tested on, and
the performance numbers measured against a representative 5-minute play session.

## Tested game data

* **USA** Xbox 360 release of *Silent Hill: Downpour*
* **Europe** Xbox 360 release of *Silent Hill: Downpour*
* Title id `4B4E0823`, base XEX hash `7A3D5809776EE6AB`
* **Title Update 1 is mandatory.** The launcher staged it interactively
  through the in-game `AcquireWizardDialog` (Skate3-style first-run flow).
  See `src/downpour_title_update_installer.{cpp,h}` for the installer logic
  and the SDK's `rex::ui::AcquireWizardDialog` for the UI.

## Test bench

* **CPU**: Intel Core i5-13400F (10 cores / 16 threads — 6P + 4E, Raptor Lake, no iGPU)
* **GPU**: NVIDIA GeForce RTX 5070 (12 GB VRAM)
* **OS**: Windows 11 Home 26200
* **Display**: 120 Hz, vsync ON, FPS limiter set to 60

## Performance results (representative session 055)

Captured 2026-06-27 21:21 with `log_to_file = true` + `log_level = warn`.
Session length: ~5 minutes, 9,069 frame-events logged.

| Metric                         | Value      | Interpretation                  |
|--------------------------------|------------|---------------------------------|
| p50 frame time                 | **16.57 ms** | 60 FPS on the median frame       |
| p90 frame time                 | 17.53 ms   | 60 FPS on 90% of frames         |
| p99 frame time                 | 30.04 ms   | ~33 FPS worst-case 1% (rare)    |
| Frames > 25 ms (sub-40 FPS)    | 1.73%      |                                 |
| Frames > 33 ms (sub-30 FPS)    | 0.63%      |                                 |
| Frames > 50 ms (visible hitch) | **0.17%**  |                                 |
| Frames > 100 ms (EXTREME)      | 0.06%      |                                 |
| Frames > 250 ms (CRITICAL)     | 0.04%      | 4 events over 5 min             |

The 4 CRITICAL spikes (250-360 ms) cluster on level transitions / scene
streaming — multi-thousand barrier transitions + ~500 memexport readback
fence waits in a single frame. These do not affect mid-scene gameplay.
See `v1.0-known-issues.md` for the open follow-up.

## Methodology

Logs are written by the SDK's `PipelineCache::ReportFrameBoundary` at every
frame closure when `log_level >= warn`. Each event records:

* `frame_ms` — wall-clock since previous closure (= the actual time you saw
  before next present)
* Render-thread buckets — `sync_compile_ms`, `block_wait_ms`, `submit_ms`,
  `present_ms`, `tex_req_ms`, `rt_resolve_ms`, `submit_barriers_ms`,
  `fence_wait_ms` (split into `qops` + `submission` sub-buckets in v1.0)
* Guest-CPU buckets (NEW in v1.0) — `guest_cpu_io_ms` (NtCreate/Read/Write/etc.),
  `guest_cpu_alloc_ms` (RtlAllocateHeap), `guest_cpu_wait_ms` (NtWait*).
* `unaccounted_ms = frame_ms - max(render_thread_accounted, guest_cpu_total)`

Severity labels: SEVERE > 50 ms, EXTREME > 100 ms, CRITICAL > 250 ms.
MILD (> 25 ms) and HIGH (> 33 ms) are gated behind `pso_stall_log_verbose`.
A new label `guest-cpu-stall` fires when `guest_cpu_total_ms > 5.0` even
on otherwise-quiet frames.

The analysis script is at `https://github.com/LittleBitUA/DownpourRecomp` and
mirrors of the canonical session-055 numbers above were generated with it.

## How to reproduce the measurement

1. In the launcher's **Debug** tab, set `Write Log File = on` and
   `Log Level = Warn`. Save.
2. Launch the game, play a representative 5+ minute segment.
3. Quit cleanly. Logs land in `logs/downpour_NNN.log` (plus rotated
   `downpour_NNN.[1-5].log` files).
4. Grep for `PSO_STALL` events. Each event has the buckets above. Drop
   them into a spreadsheet or use the included Python analyzer.
